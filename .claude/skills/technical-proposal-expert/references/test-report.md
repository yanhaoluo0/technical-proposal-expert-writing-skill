# 测试报告场景指引

本指引与 SKILL.md 中的通用写作要求同时生效。仅当匹配到「测试报告」场景时加载。

## 场景定位

适用于测试报告、测试总结、测试说明文档等。需体现测试范围、方法、结果与结论，数据与结论用叙述串联，避免纯列表堆砌。

## 建议结构

1. **概述**：测试目的、范围、环境、参与方、报告读者。  
2. **测试策略与方法**：测试类型（功能/性能/安全等）、测试方法、用例设计思路、通过/失败准则。  
3. **测试执行与结果**：执行概况、通过率与主要结论；重要缺陷或风险可单独成段说明。  
4. **结论与建议**：总体结论、遗留风险、改进或后续测试建议。

正文以叙述为主，用逻辑连接词串联。数据可用表格（如用例统计、缺陷分布），表格前后用文字归纳含义。禁词与去列表化遵循 SKILL.md 通用要求。

## 输出控制

每次输出完整的一节。长文档每约 1000 字（中文字符）暂停并等待用户确认。生成到文件时增量追加写入。

## 场景初始化话术

可向用户说明：已匹配**测试报告**场景，将按测试报告结构协助撰写；请提供测试范围、已有数据或希望先写的章节。
