# 场景索引与匹配表

匹配到场景后，**必须读取**本目录下对应的 `.md` 文件并按其中指引执行。通用写作要求见主 SKILL.md。

## 场景列表

| 场景标识 | 文件名 | 适用文档类型 |
|----------|--------|----------------|
| technical-proposal | technical-proposal.md | 技术标书、投标书、招标技术方案、技术方案书 |
| design-report | design-report.md | 方案设计报告、技术方案设计、系统设计说明 |
| test-report | test-report.md | 测试报告、测试总结、测试文档 |
| weekly-report | weekly-report.md | 周报、工作周报、周总结 |
| general | general.md | 常规文档（兜底：无法匹配时使用） |

## 匹配规则

1. 根据用户对「写什么文档」的回答，用下表关键词匹配。
2. 匹配到则加载对应文件；**无法匹配时一律使用 general.md（常规）兜底**。
3. 同一对话中一旦确定场景，后续写作均按该场景执行，除非用户明确要求切换。
