# 方案设计报告场景指引

本指引与 SKILL.md 中的通用写作要求同时生效。仅当匹配到「方案设计报告」场景时加载。

## 场景定位

适用于技术方案设计报告、系统设计说明、架构设计文档等。侧重把设计决策、技术选型、架构与模块划分、关键流程说清楚，语气专业、条理清晰，避免堆砌列表。

## 建议结构

1. **概述与目标**：方案背景、设计目标、适用范围、读者对象。  
2. **总体架构**：整体架构说明、技术栈与选型依据、模块划分与边界。  
3. **详细设计**：核心模块/子系统的设计说明（接口、数据、流程、关键算法或策略）。  
4. **非功能与约束**：性能、安全、可扩展性、部署与运维考虑（若适用）。  
5. **附录**：术语、引用、变更记录等（按需）。

正文以叙述性段落为主，必要时用「第一…在此基础上…最终」等连接。可采用适度层级（一、(一)、1.、(1)），但避免大段纯 bullet。表格可用于接口说明、参数定义、配置项等。

## 输出控制

每次输出完整的一节或一个子模块设计说明。长文档每约 1000 字（中文字符）暂停，等待用户确认后再继续。生成到文件时采用增量追加，不一次性写入全文。

## 场景初始化话术

可向用户说明：已匹配**方案设计报告**场景，将按设计报告结构协助撰写；请补充当前已有的提纲或希望先写哪一部分。
