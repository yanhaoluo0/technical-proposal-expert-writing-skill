# 周报场景指引

本指引与 SKILL.md 中的通用写作要求同时生效。仅当匹配到「周报」场景时加载。

## 场景定位

适用于工作周报、周总结、周度汇报等。内容简洁、条理清楚，以叙述概括为主，避免冗长列表和 AI 味表述。

## 建议结构

1. **本周概要**：一两段概括本周工作重点与主要成果。  
2. **完成事项**：按工作类型或项目分块，用简短段落说明完成了什么、结果如何；可适度用「第一…此外…」等连接，避免大段 bullet。  
3. **进行中/待办**：当前进行中的事项与下周计划，同样以段落形式概括。  
4. **问题与风险**（可选）：需要协调或关注的问题，简要说明。

语气偏正式但不必过于公文腔；篇幅按实际需要控制。禁词表与去列表化原则仍适用，可适当放宽对「列表」的约束（如周报中少量要点列举），但整体仍以连贯叙述为主。

## 输出控制

周报通常较短，可一次完成；若用户要求分节撰写，则每完成一节或约 1000 字暂停确认。写入文件时增量追加。

## 场景初始化话术

可向用户说明：已匹配**周报**场景，将按周报结构协助撰写；请说明本周主要工作或提供要点，我将整理成连贯的周报内容。
